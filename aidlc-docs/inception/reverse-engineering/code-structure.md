# Code Structure

## Build System

- **Type**: Makefile + Go modules + npm
- **Configuration**:
  - `Makefile` - build, deploy, and release commands
  - `go.mod` - Go module definition (Go 1.25)
  - `webapp/package.json` - npm package management
  - `webpack.config.js` - frontend bundle configuration
  - `plugin.json` - plugin manifest

## Key Classes/Modules


## Design Patterns

### Plugin Interface Pattern
- **Location**: `server/plugin.go`
- **Purpose**: Implements the Hook interface required by the Mattermost Plugin SDK
- **Implementation**: Embeds `plugin.MattermostPlugin` to inherit standard hooks, overrides required hooks

### Repository Pattern (KV Store)
- **Location**: `server/store/kvstore/`
- **Purpose**: Abstracts data access logic and improves testability
- **Implementation**: `KVStore` interface + `Client` implementation + test mocks

### Strategy Pattern (Command)
- **Location**: `server/command/command.go`
- **Purpose**: Separates command handling and improves testability
- **Implementation**: `Command` interface + `Handler` implementation + mock

### Configuration Guard Pattern
- **Location**: `server/configuration.go`
- **Purpose**: Thread-safe configuration management for concurrent access
- **Implementation**: `sync.RWMutex` + Clone pattern
