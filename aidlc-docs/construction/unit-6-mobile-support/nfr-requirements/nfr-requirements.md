# Unit 6: Mobile Support — NFR Requirements

> **Updated 2026-03-31**: The `server/push/` package has been REMOVED. Mobile clients receive call notifications via WebSocket events instead of push notifications. All NFR requirements below are no longer applicable. This document is retained for historical reference.

## Performance (REMOVED)

| Requirement | Value | Rationale |
|---|---|---|
| Push dispatch overhead per CreateCall | < 50ms for typical DM (2 members) | Sequential, max 8 calls to SendPushNotification |
| Max members dispatched | 8 | Aligns with Mattermost Calls plugin limit |
| Concurrency model | Sequential (no goroutines) | Simplicity; DM/GM channels have at most 8 members |

No pagination loop — single `GetChannelMembers(page=0, perPage=8)` call.

---

## Reliability

| Requirement | Value |
|---|---|
| CreateCall failure on push error | No — best-effort |
| endCallInternal failure on push error | No — best-effort |
| Per-member failure handling | Log warn, continue loop |
| Channel type guard | Skip non-DM/GM silently (return nil) |

---

## Testability

| Requirement | Approach |
|---|---|
| PushSender interface mockable | Yes — `server/push/mocks/mock_push.go` generated by mockgen (`go.uber.org/mock/mockgen`) |
| Sender unit tests | Use `plugintest.API` (consistent with existing `calls_test.go` pattern) |
| Test coverage targets | `SendIncomingCall`: DM/GM send, non-DM/GM skip, member skip (caller), per-member error handling |
| | `SendCallEnded`: same scenarios |

---

## Maintainability

| Requirement | Approach |
|---|---|
| Package isolation | `server/push/` — standalone package, no circular deps |
| Dependency injection | `plugin.API` injected via `NewSender(api plugin.API)` |
| Interface separation | `interface.go` + `push.go` — consistent with `server/rtkclient` pattern |
| Logging | `p.api.LogWarn` on all error paths; no sensitive data logged |

---

## Security

All SECURITY rules assessed in `business-rules.md` (Security Compliance Summary).
No blocking findings. Key points:
- No PII or tokens in log output (SECURITY-03)
- Push targets are channel members only, no user-controlled routing (SECURITY-08)
- All external API calls have error handling (SECURITY-15)

---

## Constraints

- Push is only sent for DM and GM channel types (BR-P02)
- Maximum 8 recipients per event (BR-P03)
- No new Go dependencies required
- No new HTTP endpoints
- No new KVStore keys
