## Pattern 2: Channel Type Guard Pattern

**Applies to**: `SendIncomingCall`, `SendCallEnded`

**Problem**: Push notifications for incoming calls are only meaningful in DM/GM contexts.

**Solution**: Early return for non-DM/GM channels before any API calls.

```
channel, appErr := p.api.GetChannel(session.ChannelID)
if appErr != nil {
    return fmt.Errorf("push: GetChannel failed: %w", appErr)
}
if channel.Type != model.ChannelTypeDirect &&
   channel.Type != model.ChannelTypeGroup {
    return nil // not a DM or GM — skip silently
}
```

## Pattern 3: Dependency Injection Pattern

**Applies to**: `Sender` struct, `Plugin` struct

**Problem**: `Sender` needs `plugin.API` access; must be testable with mock API.

**Solution**: `plugin.API` is injected at construction time.

```go
// server/push/push.go
type Sender struct {
    api plugin.API
}

func NewSender(api plugin.API) *Sender {
    return &Sender{api: api}
}
```

```go
// server/plugin.go — OnActivate
p.pushSender = push.NewSender(p.API)
```

## Pattern 4: Interface Segregation for Testability

**Applies to**: `PushSender` interface, `Plugin` struct

**Problem**: `Plugin.pushSender` must be replaceable with a mock in unit tests.

**Solution**: `Plugin` holds the `PushSender` interface, not the concrete `*Sender`.

```go
// server/plugin.go
type Plugin struct {
    ...
    pushSender push.PushSender
}
```

Mock is generated by mockgen:
```
//go:generate mockgen -destination=mocks/mock_push.go -package=mocks github.com/kondo97/mattermost-plugin-rtk/server/push PushSender
```

In tests:
```go
import mocks "github.com/kondo97/mattermost-plugin-rtk/server/push/mocks"

mockPush := mocks.NewMockPushSender(ctrl)
p.pushSender = mockPush
```

## Pattern 5: Structured Logging (No PII)

**Applies to**: All error paths in `Sender`

**Rule**: Log only `user_id`, `call_id`, `channel_id`, and error message.
Never log push notification content, usernames, or channel names.

```go
p.api.LogWarn("push: SendPushNotification failed",
    "call_id", session.ID,
    "user_id", member.UserId,
    "err", appErr.Error())
```
